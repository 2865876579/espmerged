# 项目规则

## 代码修改流程

**每次修改代码前，必须先阅读 `.claude/skills/code-check/SKILL.md`**，严格遵循四步流程：

1. **扫描分析** — 阅读所有相关文件，参考 78/xiaozhi-esp32 最佳实践
2. **列出 Bug 清单** — 按严重程度排序，用表格格式，只列不改
3. **等待确认** — 用户回复"可以"/"修复全部"/指定编号后再动手
4. **逐项修复** — 改完一个让用户确认

### 永不跳过

- 哪怕只有一个 bug，也要先列表
- 火山引擎/第三方 API 接入也是：先查文档 → 列方案 → 等确认
- 调试时发现的缺失字段、语法错误，同样先报再改
